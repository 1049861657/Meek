# 迁移交付

> 流程见 [SKILL.md](./SKILL.md)。**读者 = 执行本批的父 agent**。门禁 subagent · proposals 裁决 · 交付模板。

## 硬规则

- 门禁：`generalPurpose` + `readonly`，**禁 Bugbot**
- Findings → BLOCK；proposals **不** BLOCK
- 审阅范围 =「本批变更清单」路径，**禁**工作区全量 diff
- **勾选任务书 ≠ 交付**；PASS 后须**逐条裁决** proposals（含「无」）并完成对应动作，方能对用户说完成

## 交付前自检（逐项打勾，缺一即不得交付）

1. `pnpm build` 通过
2. 本批 `todos/M*.md` 父任务已 `[x]`
3. `todos/README.md` 进度已更新
4. 门禁 Verdict = PASS
5. 裁决表已填写（proposals 为「无」也占一行）
6. 凡终裁「写入 todos」：`Grep` 本批新增的 `Mx-yy-Dnn` **全部为 `[x]`**，**零条** `- [ ]`
7. 裁决表「落点」列：**无**「下批」「M6 再做」「M6 批次」等推迟用语
8. 用户回复含完整裁决表

## 变更清单（启门禁前，边编码边记）

```markdown
Batch: M3-04 | 新增/修改/删除/不属于本批/超任务书（无则写无）
| 路径 | 摘要 |
```

## 门禁 subagent

`generalPurpose` · `readonly` · `run_in_background: false` · `Migrate delivery gate`

**Prompt**（填实参路径）：

```text
Meek: … | 参考: … | Batch: …
本批变更清单: <上表>
Read: delivery.md, optimize.md

任务 A — 仅清单内文件，对照参考 + routes.ts + storage-contract + README 禁止项 + 任务书；超任务书已落地无说明 → BLOCK。
核对：①参考映射 ②无禁项新能力 ③API/存储键 ④清单=实现 ⑤build ⑥todo 勾选一致 ⑦未授权 optimize。

任务 B — 不 BLOCK：| # | 类型 | 建议 | 理由 | 影响 | 建议处置 |（本批采纳|写入 todos|拒绝）

输出：## Verdict ## Findings ## Optimization proposals（无则写无）## Notes
```

| Verdict | 父 agent 下一步 |
|---------|----------------|
| BLOCK | 修 Findings → build → 重跑门禁（≤2 次） |
| PASS | 按下方「PASS 后逐步执行」走完再交付 |

## PASS 后逐步执行（父 agent）

```
1. 读子 agent 的 Optimization proposals（无则裁决表写一行「无」）
2. 对每条 proposal 选一种终裁（见下节决策树），禁止原样抄「建议处置」
3. 按终裁立即改文件 / 勾选 todo（禁止只写裁决表不动手）
4. Grep 本批 `Mx-yy-Dnn`：凡「写入 todos」须全为 `[x]`
5. 填用户回复（含裁决表 + 优化差异）
```

## 终裁决策树（先判断，再动手）

```
proposal 本批能否立即完成？
├─ 能，且只需改正文/代码，不必在任务书留 Dnn 子项
│    → 本批采纳
├─ 能，且须在任务书留 Dnn 子项（通常含编码）
│    → 写入 todos（追加子项 + 同批完成 + 同批 [x]）
└─ 不能，明确归属后续里程碑（例 M6-02 运行时补验）
     → 拒绝（写清归属 Mx-yy）；禁止标「写入 todos」
```

**跨文件不算推迟理由**：proposal 指向 `todos/M6-*.md` 但改动只是补路径、改数字、改措辞 → 仍属「本批采纳」，当场改文件。

## 三种终裁（定义 + 必做动作）

子 agent「建议处置」≠ 终裁。`无` / `已跳过` 也须在裁决表占一行。

### 本批采纳

| 项 | 说明 |
|----|------|
| 何时选 | 本批内可立即落地；含 **任意 `todos/*.md` 的文案修正**（笔误、路径、数字），**与 proposal 写在哪个 M*.md 无关** |
| 必做 | 当场改对应文件；在用户回复「优化差异」列出路径 |
| 裁决表落点 | 已修改的文件路径（例 `todos/M6-hardening.md`） |
| 禁止 | 标「写入 todos」却不动文件；写「下批再做」 |

### 写入 todos

| 项 | 说明 |
|----|------|
| 何时选 | 须在**本批父任务**下留 `Mx-yy-Dnn` 子项，且**本批就要编码完成** |
| 必做 | ① 父任务下追加 `- [ ] **Mx-yy-Dnn** …` ② 编码 ③ 改 `- [x]` ④ `Grep` 确认无未勾子项 |
| 裁决表落点 | 已 `[x]` 的子项全文或 `文件:行` |
| 禁止 | 单开 `### 门禁待办`；留 `- [ ]`；落点写「M6 批次」「执行时」 |

### 拒绝

| 项 | 说明 |
|----|------|
| 何时选 | 归属其他里程碑，**本批明确不做**（例 M4 未就绪时的运行时补验） |
| 必做 | 裁决表写理由 + 归属 `Mx-yy` |
| 禁止 | 用「写入 todos」代替拒绝 |

## 终裁约束

- 触 optimize 禁止项 / API·存储键 / BullMQ → 禁「本批采纳」
- 子 agent 写「写入 todos」+「下批」→ 父 agent **改裁**为「拒绝」或「本批采纳」，**禁**原样交付
- 用户「仅 migrate」→ 全「拒绝」或「写入 todos」本批完成，**仍填裁决表**

## 反例（M3-08，禁止重演）

| proposal | 错误裁决 | 正确裁决 |
|----------|----------|----------|
| M6-02-02「6 Modal」→「9 Modal」 | 写入 todos（M6 批次） | **本批采纳** → 当场改 `M6-hardening.md` |
| M6-02-04 补 gate 路径 | 写入 todos（M6 执行时） | **本批采纳** → 当场改 `M6-02-04` 文案 |
| M4 门控四项运行时补验 | 写入 todos（下批） | **拒绝**，归属 `M6-02-04` / `M4` |

## 用户回复（PASS 后）

做了什么 · 本批变更表 · 参考路径 · 优化差异（无则 parity migrate）· **优化建议（门禁）**：

`| # | 建议 | 终裁 | 落点 |`

- 「本批采纳」落点 = 已改文件路径
- 「写入 todos」落点 = 已 `[x]` 的 `Mx-yy-Dnn` 子项
- 「拒绝」落点 = 归属 `Mx-yy` + 一句理由

末尾：`门禁 PASS`
